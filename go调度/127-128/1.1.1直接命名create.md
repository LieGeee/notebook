```go
type ShortLinkRepository interface {

    Create(ctx context.Context, link *model.ShortLink) error

    FindByCode(ctx context.Context, code string) (*model.ShortLink, error)

    FindByUserID(ctx context.Context, userID int64, page, limit int) ([]*model.ShortLink, int64, error)

    Delete(ctx context.Context, code string) error

}

```
在 **Go 语言的惯用规范（Idiomatic Go）** 和 **Repository 模式** 中，直接命名为 `Create` 其实是**最推荐**的做法。

我来解释一下为什么这样命名反而更好，以及什么时候才需要改名。

### 1. 为什么 `Create` 是好的命名？

#### A. 上下文决定含义（Context is Key）

在 Go 语言中，我们极力避免“口吃”（Stuttering）现象，即重复命名。 这个接口叫 `ShortLinkRepository`。当你调用它的时候，代码是这样的：

Go

```go
// 假设你实例化了一个变量叫 shortLinkRepo
shortLinkRepo.Create(ctx, link)
```

读起来就是：“**短链仓库 -> 创建**”。语意非常通顺。

如果你把它命名为 `CreateShortLink`，调用起来就会变成：

Go

```go
shortLinkRepo.CreateShortLink(ctx, link) 
// 读起来是：“短链仓库 -> 创建短链”。
// "ShortLink" 出现了两次，这就叫“口吃”，显得啰嗦。
```

#### B. 参数类型已经提供了“关联性”

看一下你的函数签名：

Go

```go
Create(ctx context.Context, link *model.ShortLink) error
```

参数 `link` 的类型是 `*model.ShortLink`。这已经通过**强类型**约束了关联性。你不可能把一个 User 对象传给这个 Create 方法，编译器会报错。所以方法名不需要再强调“我是创建短链”。

---

### 2. 什么时候不该用 `Create`？

只有在以下情况，你才需要让名字更具体：

1. **一个 Repo 处理多种关联实体（不推荐，但存在）**： 如果你的 Repository 既负责存短链，又负责存短链的访问日志（AccessLog），那你就不能只叫 Create 了。

    ```go
    // 不好的设计：一个 Repo 管太多事
    type MixedRepository interface {
        CreateShortLink(...)
        CreateAccessLog(...) // 这时才需要区分
    }
    ```
    
    _改进建议：拆分成 `ShortLinkRepo` 和 `AccessLogRepo`，然后各自都叫 `Create`。_
    
2. **有特殊的业务语义**： 如果只是单纯的“插入数据库”，用 `Create` 或 `Insert`。 如果是“创建并发布”或者“创建如果存在则更新”，可以用更具体的词：

  - `Save` (通常暗示：没有就新增，有就更新)

   - `Upsert` (更新或插入)
  
  - `BatchCreate` (批量创建)


---

### 3. 常见的替代方案

如果你还是觉得 `Create` 太干巴巴，只有这几个词是符合 Go 规范的替代品：

- **`Save`**: `repo.Save(ctx, link)` —— 很多 ORM（如 GORM）习惯用 Save。这通常给人一种“把这个对象持久化”的感觉，不管它是新的还是旧的。

- **`Insert`**: `repo.Insert(ctx, link)` —— 这个词比 Create 更底层一点，明确表示是 SQL 的 INSERT 操作。


### 总结建议

**保持 `Create` 不变，这是最地道的写法。**

在 Go 的 Interface 设计中：**接口名负责“名词”（谁），方法名负责“动词”（做什么）。**
- **Interface**: `ShortLinkRepository` (我是管短链的)
- **Method**: `Create` (我负责创建)
组合起来就是 `ShortLinkRepository.Create`，清晰且简洁。
